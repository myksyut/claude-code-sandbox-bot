# プロジェクトコンテキスト

## プロジェクト概要

- **解決する問題**: Slackから直接Claude Codeを呼び出し、組織のGitHubリポジトリに対するコードベース調査やMCPタスクを実行できる環境がない
- **ターゲットユーザー**: 個人利用（PoC検証段階）
- **利用シーン**: Slackチャンネルからボットをメンション→クラウド上のサンドボックスでClaude Codeが起動→リポジトリをクローンして調査→結果をSlackに返信

## 開発体制

- **チーム構成**: 個人開発
- **開発フェーズ**: プロトタイプ（動作検証優先）

## ビジネス制約

1. **リトライ可能な設計**: 各処理は冪等性を担保し、失敗時に再実行できること
2. **中間状態管理**: 実行中の状態をRedisで管理（揮発性OK、永続化不要）
3. **Human-in-the-loop対応**: 不確実な出力は人間確認フローを経由させる設計を考慮すること

## システムの主要機能

1. **Slack Bot**: ユーザーからのメンション・コマンドを受信
2. **Claude Code Sandbox**: クラウド上で隔離されたClaude Code実行環境を提供
3. **GitHub連携**: 組織のリポジトリを任意にクローン可能
4. **MCP統合**: 汎用タスク実行のためのMCPサーバー設定
5. **Skills/Agent設定**: コードベース調査を効率化するカスタム設定
